Highlights

  • Module is container of reusable components, it is defined by using angular.module()
  • One html page can be assigned only one module, via ng-app directive on html tag
  • app.js is the entry point of the js part of the application
  • ng-include is used to load html fragements from external html file, it limits the file to be in the same domain
  • Auto-bootstraping process
    1. Angular initializes automatically upon DOMContentLoaded event or when the angular.js script is downloaded to the browser and the document.readyState is set to complete.
    2. AngularJS looks for the ng-app directive
    3. After finding the ng-app directive
      1. Load the module associated with the directive.
      2. Create the application injector.
      3. Compile the DOM starting from the ng-app root element.

Highlights

  • AngularJS controllers are just JavaScript constructor functions that are used to hold scopes and to expose behaviors that views can use
  • The controller defines the actual behavior of your app. It contains business logic for the view and connects the model to view with the help of $scope.
  • A controller is associated with a HTML element with the ng-controller directive.
  • When a controller is created during runtime, the $injector will inject the $scope argument
  • Controllers add properties and functions to the $scope, in this example, we have property displayDefaultContent and function toggleDefaultContent, and they can be accessed in the expressions assigned to ng-click and ng-show.
  • ng-click and ng-show are built-in directives, which we will cover soon
